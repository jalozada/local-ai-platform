# ES-003: Architecture Before Implementation

## Status

Accepted

## Purpose

Ensure implementation work is guided by clear architectural intent before technical details are introduced.

## Standard

Significant platform changes should be preceded by architecture review, documented intent, and clear boundaries. Implementation should follow approved architecture rather than define architecture implicitly through code or configuration.

## Rationale

Architecture-first development improves coherence, limits accidental complexity, and makes long-term platform evolution easier to manage.

## Exceptions or Revisit Conditions

Small documentation corrections, low-risk maintenance, or exploratory work may proceed without formal architecture updates. This standard should be revisited if the review process becomes too heavy for the pace of development.
