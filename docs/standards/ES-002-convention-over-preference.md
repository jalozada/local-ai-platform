# ES-002: Convention Over Preference

## Status

Accepted

## Purpose

Establish consistent repository practices that reduce subjective variation and improve maintainability.

## Standard

Prefer documented project conventions over individual preference. Naming, structure, documentation style, and workflow patterns should follow established repository standards unless there is a clear engineering reason to diverge.

## Rationale

Consistent conventions make the repository easier to review, extend, and operate. They reduce unnecessary debate and help future contributors understand the platform without relying on personal context.

## Exceptions or Revisit Conditions

Exceptions are allowed when an existing convention creates technical risk, blocks an approved requirement, or no longer fits the platform's architecture. In those cases, the convention should be revisited and documented.
