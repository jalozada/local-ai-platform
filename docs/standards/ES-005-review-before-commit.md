# ES-005: Review Before Commit

## Status

Accepted

## Purpose

Promote disciplined change review before work is committed to the repository history.

## Standard

Changes should be reviewed for scope, correctness, documentation impact, and alignment with approved architecture before commit. Work orders should be completed with a clear summary of created, updated, or deferred items.

## Rationale

Review before commit keeps repository history intentional and reduces the likelihood of committing incomplete, unrelated, or poorly documented changes.

## Exceptions or Revisit Conditions

Urgent recovery work or temporary local checkpoints may bypass normal review expectations. The resulting changes should be reviewed and cleaned up before becoming part of approved project history.
