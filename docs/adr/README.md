# Architecture Decision Records

## Purpose

Architecture Decision Records capture significant engineering decisions that shape the platform. They preserve the context, decision, and consequences behind choices that affect architecture, operations, or long-term maintainability.

## When to Create an ADR

Create an ADR when a decision affects platform direction, introduces or rejects major technology choices, changes architectural boundaries, or creates long-term operational consequences.

## Expected Format

Each ADR should include:

- Title
- Status
- Context
- Decision
- Consequences

Use `ADR-TEMPLATE.md` as the starting point for new records.
